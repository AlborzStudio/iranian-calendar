# Contributing to Iranian Calendar

Thank you for your interest in contributing to the Iranian Calendar project!

## Attribution Requirements

**IMPORTANT:** This project is built on the theoretical work of **Sherwin Vakili** and implemented by **Alborz Teymoorzadeh**. All contributions must maintain proper attribution to both authors.

Any derivative work, fork, or modification MUST:
1. Credit **Sherwin Vakili** for the theoretical framework
2. Credit **Alborz Teymoorzadeh** for the original implementation
3. Maintain the AUTHORS.md file
4. Include attribution in any documentation

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:
1. Check existing issues first
2. Open a new issue with a clear description
3. Include steps to reproduce (for bugs)
4. Provide expected vs actual behavior

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature`
3. **Make your changes**
4. **Run tests**: `python ic/tests/run_tests.py`
5. **Commit with clear message**: `git commit -m "Add feature: description"`
6. **Push to your fork**: `git push origin feature/your-feature`
7. **Open a Pull Request**

### Code Style

- Follow PEP 8 Python style guidelines
- Use type hints where appropriate
- Add docstrings to new functions
- Write tests for new features
- Keep functions focused and modular

### Testing

All contributions must include tests:
```bash
# Run all tests
python ic/tests/run_tests.py

# Tests must pass on Python 3.8+
```

### Documentation

- Update README.md if adding new features
- Add docstrings to new functions
- Update CHANGELOG.md with your changes

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Focus on the code, not the person
- Give credit where credit is due

## Questions?

Open an issue for questions or discussions!
